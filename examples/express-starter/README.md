# Zustand Multiplayer Express Starter Example

This example demonstrates how to use the `@hpkv/zustand-multiplayer` middleware with an Express.js backend and a simple HTML/JavaScript frontend.

## Features

- Real-time synchronization of a todo list across multiple clients.
- Express.js server for:
  - Serving the static HTML, CSS, and JavaScript frontend.
  - Providing an API endpoint (`/api/hpkv-token`) for HPKV WebSocket token generation.
- Client-side state management with Zustand.
- Basic UI for adding, toggling, and deleting todos.
- Connection status indicator.

## Prerequisites

1.  **Node.js and npm/yarn**: Ensure you have Node.js (which includes npm) installed. Yarn is an alternative package manager.
2.  **HPKV Account**: Sign up for a free HPKV account at [https://hpkv.io/signup](https://hpkv.io/signup).
3.  **API Key and Base URL**: Create an API key in the [HPKV Dashboard](https://hpkv.io/dashboard/api-keys) and note your API Key and API Base URL.

## Setup

1.  **Clone the Repository (if you haven't already):**
    ```bash
    git clone <repository-url>
    cd <repository-name>/examples/express-starter
    ```

2.  **Install Dependencies:**
    Navigate to the `express-starter` directory and install the dependencies:
    ```bash
    npm install
    # or
    # yarn install
    ```

3.  **Configure Environment Variables:**
    Create a `.env` file in the `express-starter` directory by copying the example:
    ```bash
    cp .env.example .env
    ```
    Edit the `.env` file and replace the placeholder values with your actual HPKV API Key and Base URL:
    ```env
    HPKV_API_BASE_URL=YOUR_ACTUAL_HPKV_API_BASE_URL
    HPKV_API_KEY=YOUR_ACTUAL_HPKV_API_KEY
    ```

4.  **Configure Client-Side HPKV API Base URL:**
    Open `public/config.js` and replace `'YOUR_HPKV_API_BASE_URL_HERE'` with your actual HPKV API Base URL.
    ```javascript
    // public/config.js
    window.HPKV_API_BASE_URL = 'YOUR_ACTUAL_HPKV_API_BASE_URL'; // <-- Update this line
    ```
    This URL is used by the client-side store to connect to HPKV for state storage (via the token obtained from your Express server).

## Running the Example

1.  **Start the Express Server:**
    ```bash
    npm start
    # or for development with nodemon (if you want auto-restarts on file changes)
    # npm run dev
    ```
    The server will typically start on `http://localhost:3001` (or the port specified by the `PORT` environment variable).

2.  **Open in Browser:**
    Open your web browser and navigate to `http://localhost:3001`.

    You should see the todo list application. Try opening it in multiple browser windows or tabs to see the real-time synchronization in action.

## How It Works

-   **`server.js`**: This Express server sets up an API endpoint (`/api/hpkv-token`) using `TokenHelper` from `@hpkv/zustand-multiplayer/token-helper` to securely generate HPKV WebSocket tokens. It also serves static files from the `public` directory.
-   **`public/config.js`**: Configures the `HPKV_API_BASE_URL` for the client-side application. This is necessary for the client to know which HPKV instance to point to (the token itself is generated by your server).
-   **`public/scripts/store.js`**: Defines the Zustand store using the `multiplayer` middleware. It configures the `namespace`, `apiBaseUrl` (from `config.js`), and `tokenGenerationUrl` (pointing to the Express server's endpoint).
-   **`public/index.html`**: The main HTML page that structures the todo app.
-   **`public/app.js`**: Contains the client-side JavaScript logic to interact with the Zustand store (add, toggle, delete todos), render the todo list, and display the connection status.

When a user interacts with the todo list (e.g., adds a todo), the change is made to the local Zustand store. The `multiplayer` middleware then publishes this change to the HPKV service. Other connected clients subscribed to the same namespace receive this update via WebSocket and update their local stores, resulting in a synchronized UI.

## Folder Structure

```
express-starter/
├── public/
│   ├── scripts/
│   │   └── store.js       # Client-side Zustand store with multiplayer middleware
│   ├── app.js             # Client-side application logic (UI interaction)
│   ├── config.js          # Client-side configuration (HPKV API Base URL)
│   └── index.html         # Main HTML file for the todo app
├── .env                   # Server environment variables (HPKV Key & Base URL) - (You create this)
├── .env.example           # Example environment variables file
├── package.json           # Project dependencies and scripts
├── server.js              # Express server setup and token generation endpoint
└── README.md              # This file
```
